## 功能一：游戏管理

### 添加游戏

1. 点击左侧边栏底部 **首页** Tab，或点击 **+ 添加游戏** 按钮
2. 在弹出的对话框中：
   - 点击 **浏览** 选择游戏根目录（例如 `D:\Steam\steamapps\common\Valheim`）
   - 游戏名称会自动从目录名填入，也可以手动修改
   - 可选填写可执行文件名（例如 `valheim.exe`）
3. 点击 **添加** 完成，游戏会立即出现在左侧列表中

> **什么是游戏根目录？**  
> 就是包含游戏 `.exe` 文件的那个文件夹，不是 Steam 主目录。

### 切换游戏

点击左侧边栏中的游戏名称即可切换到对应游戏的详情页。

### 删除游戏

> 目前需要通过代码删除，UI 的删除按钮在后续版本加入。

## 功能二：BepInEx 安装管理

入口：游戏详情页顶部状态栏点击 **安装 BepInEx** 或 **管理**

### 检测安装状态

进入游戏详情页时自动检测：

- 检查游戏目录下是否存在 `BepInEx/core/BepInEx.Core.dll`
- 读取版本号（从 `changelog.txt` 获取）
- 状态栏显示 **已安装** / **未安装** 徽章

### 获取版本列表

BepInEx 安装页打开时自动请求：

```
GET https://api.github.com/repos/BepInEx/BepInEx/releases
```

- 自动过滤出 Windows 可用的 zip 附件
- 按架构分类显示：**x64**（64位）/ **x86**（32位）
- 显示发布日期，最多展示最近 10 个 Release

### 安装流程

1. 在版本列表中选择目标版本和架构（通常选最新的 x64）
2. 点击 **安装** 按钮
3. 实时进度条显示两个阶段：
   - **下载中**（0% → 60%）：从 GitHub 流式下载 zip 文件
   - **解压中**（61% → 100%）：将文件解压到游戏根目录
4. 完成后自动刷新安装状态

安装后游戏目录结构：
```
游戏根目录/
├── BepInEx/
│   ├── core/          ← BepInEx 核心文件
│   ├── patchers/
│   ├── plugins/       ← 放 Mod 的地方
│   └── config/        ← Mod 配置文件
├── winhttp.dll        ← Doorstop 注入器
├── doorstop_config.ini
└── 游戏.exe
```

### 卸载

点击 **卸载** 按钮：
- 删除 `BepInEx/core`、`BepInEx/patchers`、`BepInEx/cache` 目录
- 删除 `winhttp.dll`、`doorstop_config.ini` 等 Doorstop 文件
- **不会删除** `BepInEx/plugins`（你的 Mod 会保留）

## 功能三：Mod 管理

入口：点击左侧游戏名称进入详情页

### 扫描 Mod

- 点击顶部 **刷新** 按钮，或进入页面时自动扫描
- 递归扫描 `BepInEx/plugins/` 目录及其子文件夹
- 识别 `.dll`（启用）和 `.dll.disabled`（禁用）两种状态

### Mod 列表信息

每个 Mod 条目显示：
- Mod 名称（从文件名提取）
- 启用/禁用状态
- 文件名

### 启用 / 禁用 Mod

点击 Mod 条目左侧的 **开关**：
- **启用** → 将 `.dll.disabled` 重命名为 `.dll`
- **禁用** → 将 `.dll` 重命名为 `.dll.disabled`

> 禁用后游戏启动时 BepInEx 不会加载该 Mod，但文件不会删除。

### 搜索 Mod

在列表顶部的搜索框中输入关键词，实时过滤 Mod 列表。

### 删除 Mod

点击 Mod 条目右侧的 **删除图标** → 确认弹窗 → 永久删除文件。

> ⚠ 删除操作不可撤销，文件会从磁盘永久移除。

### 打开 plugins 文件夹

点击顶部 **打开目录** 按钮，用资源管理器打开 `BepInEx/plugins/` 目录，方便手动拖入 Mod 文件。

## 功能四：Mod 配置编辑

入口：Mod 列表中点击某个 Mod 右侧的 **配置图标**

### 读取配置

- 自动列出 `BepInEx/config/` 目录下所有 `.cfg` 文件
- 优先打开与当前 Mod 名称匹配的配置文件
- 如果有多个配置文件，可在顶部下拉框切换

> `.cfg` 文件在游戏第一次运行后才会生成，若为空请先启动一次游戏。

### 编辑配置

配置按 Section 分组显示，每个条目根据值类型自动渲染控件：

| 类型 | 控件 | 示例值 |
|------|------|--------|
| `boolean` | 开关 Toggle | `true` / `false` |
| `number` | 数字输入框 | `100`, `1.5` |
| `string` | 文本输入框 | `Hello` |
| `enum` | 下拉选择框 | `Info`, `Debug`, `Warning` |

每个条目还会显示来自配置文件注释的**描述说明**。

### 保存配置

点击顶部 **保存** 按钮，将修改写回 `.cfg` 文件（INI 格式）。

## 功能五：主题切换

位于自定义标题栏右侧，点击按钮循环切换三种模式：

| 模式 | 图标 | 说明 |
|------|------|------|
| 跟随系统 | 显示器图标 | 自动跟随 Windows 深色/浅色模式 |
| 浅色 | 太阳图标 | 强制浅色（白色背景） |
| 深色 | 月亮图标 | 强制深色（黑色背景） |

主题偏好会**持久化保存**到本地，重启应用后自动恢复。

## 功能六：应用设置

入口：左侧边栏底部 **设置** Tab

| 设置项 | 说明 |
|--------|------|
| BepInEx 下载源 | GitHub 官方 / 镜像源（镜像源接口待接入） |
| HTTP 代理 | 设置下载代理，例如 `http://127.0.0.1:7890` |
| 界面语言 | 简体中文 / English（国际化功能待完善） |

点击 **恢复默认** 按钮重置所有设置。

## 快速上手流程

```
1. 点击 + 添加游戏
   └─ 选择游戏目录 → 填写名称 → 添加

2. 进入游戏详情页
   └─ 状态栏显示"未安装" → 点击"安装 BepInEx"

3. BepInEx 安装页
   └─ 选择版本（推荐最新 x64）→ 点击安装 → 等待完成

4. 返回游戏详情页（点击左侧游戏名）
   └─ 状态变为"已安装" → 点击"打开目录"

5. 将 Mod .dll 文件放入打开的 plugins 文件夹

6. 回到游戏详情页点击"刷新"
   └─ Mod 列表出现 → 可启用/禁用/配置
```
